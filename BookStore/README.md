# Приложение "Книжный магазин"

Приложение для учета ассортимента книг в магазине на базе WinForms и ADO.NET.

## Функциональность

### Управление книгами
- Добавление книг
- Редактирование параметров книг
- Удаление книг

### Операции с книгами
- Продажа книг
- Списание книг
- Внесение книг в акции (со скидкой)
- Откладывание книг для конкретного покупателя

### Поиск
- По названию книги
- По автору
- По жанру

### Статистика
- Список новинок (книги текущего года)
- Список самых продаваемых книг
- Список самых популярных авторов
- Список самых популярных жанров

Статистика доступна за период:
- День
- Неделя
- Месяц
- Год

### Авторизация
- Вход по логину и паролю
- По умолчанию создается пользователь:
  - Логин: `admin`
  - Пароль: `admin`

## Структура данных

### Книга (Book)
- Название книги
- ФИО автора
- Название издательства
- Количество страниц
- Жанр
- Год издания
- Себестоимость
- Цена для продажи
- Продолжение другой книги (опционально)
- Количество на складе
- Участие в акции (да/нет)
- Размер скидки (если участвует в акции)

## Технологии

- .NET 10.0
- Windows Forms
- ADO.NET (Microsoft.Data.SqlClient)
- SQL Server LocalDB

## Требования

- .NET 10.0 SDK
- SQL Server LocalDB (обычно устанавливается вместе с Visual Studio)

## Запуск приложения

1. Убедитесь, что установлен .NET 10.0 SDK
2. Откройте проект в Visual Studio или используйте команду:
   ```
   dotnet restore
   dotnet build
   dotnet run
   ```
3. При первом запуске автоматически создастся база данных BookStore в LocalDB
4. Войдите в систему используя логин `admin` и пароль `admin`

## База данных

База данных создается автоматически при первом запуске приложения в SQL Server LocalDB.

Таблицы:
- `Books` - книги
- `Sales` - продажи
- `WriteOffs` - списания
- `Reservations` - резервирования
- `Users` - пользователи

## Примечания

- Пароли хранятся в открытом виде (для учебных целей). В продакшене необходимо использовать хеширование.
- База данных создается автоматически при первом запуске.
- Для работы приложения требуется SQL Server LocalDB.
